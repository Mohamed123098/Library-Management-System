<h2>
  Q: When should I use a struct instead of a class?
</h2>

<h4 >

<span >• It represents a value, not an object.</span><br>
<span >Like a number, coordinate, color, or date — no identity, only data matters.</span><br><br>

<span >• It is small in size.</span><br>
<span >Ideally under ~16–24 bytes.</span><br>
<span >Example: Point, Color, DateRange.</span><br><br>

<span >• It should be immutable.</span><br>
<span >The data shouldn’t change after creation.</span>

</h4>
